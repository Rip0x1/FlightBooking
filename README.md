# FlightBooking

## Описание
Проект FlightBooking представляет собой систему бронирования авиабилетов с интуитивным пользовательским интерфейсом, разработанную с использованием C# и XAML с интеграцией Material Design. Приложение позволяет управлять рейсами, бронированиями, платежами и пользователями, обеспечивая анимации, тени и удобные вкладки для навигации. Система поддерживает экспорт данных в Excel и фильтрацию по статусам. Проект включает как клиентскую часть (WPF), так и API для серверного взаимодействия, что делает его масштабируемым и удобным для расширения.

## Особенности
- Бронирование рейсов с выбором пользователя и обновлением статуса (подтверждено, ожидает, отменено).
- Управление бронями и их статусами через интерактивный интерфейс.
- История бронирований с фильтрацией по статусу и анимацией появления списка.
- Обработка платежей с поддержкой различных методов оплаты (кредитная карта, наличные, банковский перевод).
- Поиск рейсов по городу и дате.
- Генерация отчетов и управление пользователями.
- Экспорт данных в Excel для всех секций.
- Анимации и тени для улучшения пользовательского опыта.
- Интеграция с API для динамического обновления данных.
- Поддержка локализации интерфейса (на данный момент доступен русский язык).

## Структура проекта
### FlightBooking.WPF
- **Converters**: 
  - `EqualsConverter`: Преобразует равенство значений.
  - `NullToBoolConverter`: Преобразует null в булево значение.
  - `NullToVisibilityConverter`: Преобразует null в видимость элемента.
- **Models**: 
  - `Booking`: Модель бронирования с данными о рейсе, пользователе и статусе.
  - `Flight`: Модель рейса с информацией о номере, городах, времени и цене.
  - `Payment`: Модель платежа с суммой, методом и статусом.
  - `User`: Модель пользователя с именем, email и телефоном.
- **Pages**: 
  - `BookingHistoryPage`: Страница истории бронирований.
  - `BookingPage`: Страница управления и создания бронирований.
  - `FlightSearchPage`: Страница поиска рейсов.
  - `PaymentsPage`: Страница управления платежами.
  - `ReportsPage`: Страница отчетов.
  - `UsersPage`: Страница управления пользователями.
- **Services**: 
  - `ApiClient`: Сервис для взаимодействия с API.
- **ViewModels**: 
  - `BookingHistoryViewModel`: Модель представления для истории бронирований.
  - `BookingViewModel`: Модель представления для управления бронированиями.
  - `FlightSearchViewModel`: Модель представления для поиска рейсов.
  - `MainViewModel`: Главная модель представления.
  - `PaymentsViewModel`: Модель представления для платежей.
  - `RelayCommand`: Реализация команд MVVM.
  - `ReportsViewModel`: Модель представления для отчетов.
  - `UsersViewModel`: Модель представления для пользователей.
- **Commands**: Реализация команд для взаимодействия с интерфейсом.
- **MainWindow**: Главное окно приложения.
- **App**: Основной класс приложения.

### API
- **Controllers**: 
  - `BookingsController`: Контроллер для управления бронированиями.
  - `FlightsController`: Контроллер для управления рейсами.
  - `PaymentsController`: Контроллер для управления платежами.
  - `UsersController`: Контроллер для управления пользователями.
- **Data**: 
  - `FlightDbContext`: Контекст базы данных Entity Framework.
- **Models**: 
  - `Booking`, `Flight`, `Payment`, `User`: Модели данных для API.
- **Program.cs**: Точка входа в приложение API.

## Использование
1. Импортируйте SQL-файл с созданием схемы и тестовыми данными в вашу базу данных.
2. Откройте проект FlightBooking.WPF в Visual Studio и запустите клиентское приложение.
3. Запустите API-проект для обеспечения взаимодействия с сервером.
4. Используйте интерфейс для управления бронированиями, платежами, поиска рейсов, просмотра отчетов и пользователей.
5. Для экспорта данных нажмите соответствующие кнопки в интерфейсе.
6. Для изменения языка интерфейса (при добавлении локализации) настройте ресурсы в XAML-файлах.

## Дополнительная информация
- Проект оптимизирован для работы на Windows с использованием .NET Framework.
- Тестовые данные включают 10 записей для каждой таблицы (Users, Flights, Bookings, Payments) на русском языке.
- Анимации реализованы с использованием XAML-анимаций для улучшения визуального восприятия.
- Схема базы данных поддерживает связи между таблицами через внешние ключи.
- Приложение может быть расширено добавлением новых страниц или интеграцией с другими системами через API.
